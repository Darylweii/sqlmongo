# 高优先级准确性测试清单（Top 10）

> 目标：优先验证最容易影响“设备识别、时间解析、对比分析、上下文追问、分页与过滤”的链路。
> 适用入口：优先测聊天流式接口 `/api/chat/stream`，必要时补测 `/api/query`。

## 1. 单设备显式代号 + 最近一周
- 提问：`查询 a1_b9 最近一周的用电情况`
- 重点检查：设备码识别、默认数据类型、电量统计、时间范围是否正确
- 期望结果：只返回 `a1_b9`，时间落在最近 7 天，文字分析与表格设备一致
- 容易出错点：设备码被误改写、时间被扩到默认范围外

## 2. 多设备对比（英文逗号）
- 提问：`比较 a9_b6,a1_b7,a2_b14 三个设备最近一周的用电情况`
- 重点检查：多设备识别、是否进入 comparison 模式、默认展示汇总对比表
- 期望结果：分析中同时出现 3 个设备，汇总表一行一个设备，不只剩单设备数据
- 容易出错点：只取第一个设备、分页只分析当前页、汇总表退化成原始明细

## 3. 多设备对比（中文/英文分号）
- 提问：`比较 a9_b6；a1_b7;a2_b14 最近一周的用电情况`
- 重点检查：分号分隔兼容性
- 期望结果：设备列表仍然识别为 3 个设备
- 容易出错点：中文分号未切分、整句被当成一个关键词

## 4. 多设备对比（空格 / 顿号 / 连接词）
- 提问：`比较 a9_b6 a1_b7、a2_b14 和 a1_b9 最近一周的用电情况`
- 重点检查：空格、顿号、`和` 混合分隔的鲁棒性
- 期望结果：识别出全部显式设备码，设备顺序尽量与提问一致
- 容易出错点：空格未切分、连接词打断设备提取

## 5. 对比后追问，不再重复设备名
- 首轮提问：`比较 a9_b6,a1_b7,a2_b14 三个设备最近一周的用电情况`
- 二轮提问：`哪个波动更大？`
- 重点检查：上下文是否保留最近一轮关键设备与时间范围
- 期望结果：回答仍基于上轮 3 个设备，不要求用户重复输入设备名
- 容易出错点：后端只截短历史、上下文丢失、回答泛化成空泛解释

## 6. 明确画图需求追问
- 首轮提问：`比较 a9_b6,a1_b7,a2_b14 三个设备最近一周的用电情况`
- 二轮提问：`帮我画趋势图`
- 重点检查：只有明确提图时才展示图表；图表类型与问题匹配
- 期望结果：返回图表配置，前端在聊天气泡内展示趋势图/对比图，不应每次都强制出图
- 容易出错点：图表意图没传到底层、没有 `chart_specs`、追问后丢失前文设备范围

## 7. 时间语义解析
- 提问：`查询 a1_b9 本周的用电情况`
- 重点检查：`本周` 是否被正确换算为起止日期
- 期望结果：日志里的 `start_time/end_time` 与当前自然周一致
- 容易出错点：把“本周”误当最近 7 天，或结束时间截断不完整

## 8. 数值过滤
- 提问：`查询 a1_b9 最近一周功率大于100的数据`
- 重点检查：`value_filter` 是否正确下传；表格值是否都满足条件
- 期望结果：返回数据全部满足阈值条件，分析文字不再混入未过滤样本
- 容易出错点：过滤条件只出现在 LLM 文本里，没有真正进入 Mongo 查询

## 9. 分页与视图切换
- 提问：`比较 a1_b9、b1_b14、a2_b3 三个设备最近一周的用电情况，每页50条`
- 重点检查：多设备汇总表默认展示、`查看汇总 / 查看原始明细` 切换、明细分页
- 期望结果：默认先看汇总；切到原始明细后分页正常，且设备列包含多设备
- 容易出错点：多设备比较仍然被自动禁分页、切换按钮状态错乱、明细只剩单设备

## 10. 异常输入 / 无结果兜底
- 提问：`查询 x999_y999 最近一周的用电情况`
- 重点检查：无设备 / 无数据时的报错可读性
- 期望结果：明确说明未找到设备或无数据，不返回误导性统计结论
- 容易出错点：回退到错误设备、返回空表但配上正常分析结论

## 建议执行顺序
1. 先跑 1、2、5，验证当前最核心的“单设备 / 多设备 / 上下文”准确性。
2. 再跑 6、8、9，验证“图表 / 过滤 / 分页视图”交互链路。
3. 最后跑 7、10，验证时间语义和异常兜底。

## 建议记录字段
- 实际命中的设备码
- 实际 `start_time/end_time`
- `analysis.mode`
- `query_params`
- `show_table / table_type / show_charts`
- 每个 DAG / Agent 步骤耗时与总耗时
