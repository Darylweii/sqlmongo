# 混合设备解析回归清单

本文用于回归验证新设备解析主路径是否符合预期，重点覆盖两类核心行为：

- 逐目标确认：对比查询中，多个 target 必须逐个确认后再执行
- 推荐后确认不自动执行：多候选时允许推荐第一名，但不能自动代选

适用范围：

- `src/agent/dag_orchestrator.py`
- `src/agent/nodes/metadata_mapper.py`
- `src/agent/nodes/semantic_metadata_mapper.py`
- `src/agent/utils/hybrid_device_resolver.py`
- `src/metadata/metadata_engine.py`
- `web/index.html`

## 一、回归目标

### 1. 设备解析主路径

验证系统按以下顺序工作：

1. 会话记忆命中
2. 显式设备码精确匹配
3. 多字段词法召回
4. 低置信度语义补充召回
5. 融合排序
6. 自动通过 / 推荐确认 / 强制确认

### 2. 重点风险点

重点关注以下回归风险：

- 多候选时被系统偷偷自动选中
- 对比查询把整句 `A vs B` 当成一个 target 去搜
- 用户点击确认后，系统再次回到同一轮确认，形成回环
- 已确认的 target 没有继承到后续问题
- 显式设备码跨项目重码时，系统默认汇总所有同码设备
- 前端候选卡片没有展示推荐态 / 推荐来源 / 置信度

## 二、测试前准备

### 1. 环境准备

- 启动后端服务
- 启动前端页面
- 保证 MySQL / MongoDB / 设备元数据可正常访问
- 若语义索引存在，先做一轮“语义开启”验证
- 若语义索引可临时关闭，再做一轮“语义降级”验证

### 2. 每条用例记录项

建议每条测试都记录以下内容：

- 用户原始问句
- QueryPlan 的 `target / comparison_targets / data_type / range`
- 候选卡片是否出现
- 是否出现“推荐”高亮
- 点击候选后是否自动继续原问题
- 最终 `resolved_scope` 是否正确
- 是否出现重复确认 / 回环 / 自动误选

### 3. 判定标准

- 通过：行为与“预期结果”完全一致
- 可接受：结果正确，但文案/排序有轻微偏差
- 不通过：出现自动误选、重复确认、作用域错误、结果执行错误

## 三、核心回归用例

## A. 单设备唯一命中

### A1. 完整设备名直接执行

**问句**

`查询电子楼 AA2-1 八层动力今天的耗电量`

**预期**

- 不弹候选卡片
- 直接进入数据查询
- `resolved_scope` 中仅 1 个设备
- 不出现“所有同码设备”汇总选项

### A2. 唯一设备码直接执行

**问句**

`查询 b1_b14 今天的电压数据`

**预期**

- 如果库中 `b1_b14` 只有一个设备，不弹候选卡片
- 直接查询并展示数据
- 作用域卡片里该设备可显示为已确认范围
- 若该设备天然唯一，可不展示“更换设备”按钮，或显示为不可更换态

---

## B. 多候选推荐后确认

### B1. 模糊简称，多候选，必须先确认

**问句**

`查询八层动力的耗电量`

**预期**

- 弹出候选卡片
- 如果第一名明显领先，可显示“推荐”高亮
- 但不能直接执行查询
- 必须等待用户点击某个候选

### B2. 显式重码，不允许默认汇总

**问句**

`查询 a1_b9 设备今天的电流数据`

**预期**

- 因存在多个 `a1_b9`，必须弹候选卡片
- 显示多个项目下的 `a1_b9`
- 同时显示 `所有 a1_b9 设备` 选项
- 默认不能直接汇总执行
- 只有用户明确点击“选全部”后，才进入跨项目汇总

### B3. 推荐后确认不自动执行

**问句**

`查询味多美的用电量`

**预期**

- 若“味多美”匹配到多个候选，出现候选卡片
- 可出现推荐项，但不能自动进入结果页
- 用户不点击时，系统停留在待确认状态

---

## C. 对比查询逐目标确认

### C1. 两个 target 都模糊

**问句**

`对比一下电子楼八层动力和七层动力的耗电量`

**预期**

- 系统先拆分出两个 target，而不是把整句当一个 target
- 先只要求确认第一个未确认 target
- 第一个确认后，自动继续原问题
- 再确认第二个未确认 target
- 两个都确认后，才真正执行对比查询

### C2. 一个 target 已确认，一个 target 仍模糊

**前置**

- 先确认 `a1_b9 -> 1#变加联络 AA5-1 稻盛和夫`

**问句**

`a1_b9 与 a9_b2 哪个用电更多？`

**预期**

- 不再要求重新确认 `a1_b9`
- 只要求确认 `a9_b2`
- `resolved_scope` 里保留已确认的 `a1_b9`
- `a9_b2` 确认后自动继续原问题并输出对比结果

### C3. 一个 target 唯一，一个 target 多候选

**问句**

`a1_b9 与 b1_b14 哪个用电更多？`

**预期**

- 若 `b1_b14` 唯一，可直接记入已确认范围
- `a1_b9` 若多候选，必须弹确认卡片
- 用户确认 `a1_b9` 后再执行对比
- 不能先拿所有 `a1_b9` 做汇总比较

---

## D. 会话记忆继承

### D1. 已确认设备后继续追问

**前置**

- 先确认 `a1_b9 -> 中国能建集团数据机房监控项目 / B2柜`

**问句**

`它今天的电压数据`

**预期**

- 直接继承上一步已确认设备
- 不重新弹候选卡片
- 查询结果只落在该已确认设备上

### D2. 对比确认后继续追问其中一个

**前置**

- 完成 `a1_b9 与 b1_b14 哪个用电更多？` 的双目标确认

**问句**

`再看 a1_b9 今天的电流数据`

**预期**

- 若用户显式点名 `a1_b9`，应继承本会话对 `a1_b9` 的确认结果
- 不再次弹同一 alias 的确认卡片

---

## E. 重新确认与作用域重置

### E1. 重新确认单个 alias

**前置**

- 已确认 `a1_b9`

**操作**

- 点击作用域卡片上的“更换设备” / “重新确认 a1_b9”

**预期**

- 只清除 `a1_b9` 这个 alias 的会话记忆
- 其他已确认设备保留
- 重新出现 `a1_b9` 候选卡片

### E2. 清空整个会话范围

**前置**

- 已确认多个设备

**操作**

- 点击“清除当前确认范围”

**预期**

- 清空整个会话作用域
- 后续再次问 `a1_b9` 时，要重新触发确认流程

### E3. 跨项目汇总态提示

**操作**

- 在 `a1_b9` 候选卡片中点击 `所有 a1_b9 设备`

**预期**

- 成功进入跨项目汇总查询
- 作用域卡片明确显示“当前为跨项目汇总作用域”
- 作用域卡片包含汇总代号信息

---

## F. 语义补充召回与降级

### F1. 省略表达，词法弱命中，语义补充生效

**问句**

`对比一下大厅照明和八层动力的耗电量`

**预期**

- 若词法结果不稳定，可补语义候选
- 候选卡片中可显示 score
- 但仍必须由用户确认，不能直接执行

### F2. 语义索引不可用时降级

**前置**

- 临时禁用或移走语义索引

**问句**

`查询八层动力的耗电量`

**预期**

- 系统仍能通过增强词法召回给出候选
- 不因语义索引不可用而报错
- 候选流程和确认流程仍可正常工作

---

## G. 候选卡片前端展示

### G1. 推荐态展示

**问句**

`查询八层动力的耗电量`

**预期**

- 第一名若为推荐项，应有明显“推荐”标签
- 卡片应比普通候选更突出

### G2. 匹配来源与置信度展示

**问句**

`查询味多美的用电量`

**预期**

- 候选卡片可以看到类似：
  - `词法召回`
  - `语义召回`
  - `融合排序`
  - `高置信 / 中置信 / 低置信`

### G3. 显式码冲突与 score 展示规则

**问句**

`查询 a1_b9 今天的电量`

**预期**

- 显式设备码冲突候选显示 `精确码命中`
- 不展示 score
- 语义模糊候选才显示真实 score

---

## H. 防回环验证

### H1. 点击候选后自动继续，但不能再次要求确认同一 alias

**问句**

`查询 a1_b9 设备 2024年1月的电量数据，每页50条`

**操作**

- 选择某一个具体 `a1_b9`

**预期**

- 页面出现“已确认：a1_b9 -> 正式设备，继续查询原问题”
- 随后直接进入真实查询
- 不能再次弹出同一组 `a1_b9` 候选

### H2. 对比查询逐个确认不回环

**问句**

`a1_b9 与 b1_b14 哪个用电更多？`

**操作**

- 先确认 `a1_b9`

**预期**

- 系统进入对 `b1_b14` 的确认或直接继承唯一结果
- 不应再次回到 `a1_b9` 候选列表

## 五、建议优先回归顺序

如果只做一轮最关键的 smoke，建议按下面 8 条执行：

1. `查询电子楼 AA2-1 八层动力今天的耗电量`
2. `查询八层动力的耗电量`
3. `查询 a1_b9 设备今天的电流数据`
4. `a1_b9 与 b1_b14 哪个用电更多？`
5. `对比一下电子楼八层动力和七层动力的耗电量`
6. `查询味多美的用电量`
7. `查询 a1_b9 设备 2024年1月的电量数据，每页50条`
8. 点击一次 `所有 a1_b9 设备`，验证跨项目汇总作用域提示

## 六、通过标准

本轮回归建议以以下标准作为“可发布”：

- P0：不能出现自动误选、多候选直接执行、逐目标确认回环
- P0：点击候选后必须能自动继续原问题
- P1：推荐态、来源、置信度、汇总态提示应显示正确
- P1：会话继承和单 alias 重置行为正确
